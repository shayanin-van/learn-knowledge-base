# Session Log — 2026-05-09

## What Was Discussed

### Project scope and context
Full project context was established for the first time:
- **Company**: Thai EdTech, online learning platform
- **Users**: ~70% high school students (O-NET/PAT/TCAS prep), rest lower secondary and primary
- **Goal**: AI companion for homework help and exam prep — must be trustworthy and pedagogically sound
- **Language**: Thai (primary); IPST/สสวท. as authoritative curriculum standard; international sources as enrichment
- **Phase 1 content**: High school math, physics, biology, chemistry
- **Project owner's role**: Build the knowledge base layer only; separate dev team builds the application

### Architecture decision
Discussed and decided on a **hybrid architecture**:
- **Layer 1**: Curated concept pages (LLM Wiki style), organized by curriculum topic, IPST-authoritative, multi-level explanation, cross-linked
- **Layer 2**: RAG vector retrieval operating on the curated pages (not raw PDFs)
- **Rationale**: Pure RAG too risky for high-stakes content; pure wiki too static for diverse sources. Quality from curation, discoverability from RAG.

### Notable details
- Pedagogy should be configurable per student — Socratic vs. direct. Knowledge base must be rich enough to support both.
- Interactive simulations are planned (project owner will code them) — need a linking strategy in the concept page format.
- Textbooks and past exam papers are already accessible.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Hybrid architecture: curated wiki + RAG | Trustworthiness requires curation; scale requires semantic retrieval |
| IPST as authoritative source | Thai national curriculum; must win when sources conflict |
| Pedagogy is an application-layer concern | Knowledge base should be rich enough to support multiple teaching styles |
