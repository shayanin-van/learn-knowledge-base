# Session: 2026-05-09 (Session 2)

## What Was Discussed

Architecture re-evaluation requested by project owner after reflecting on the initial architecture decision. Owner wanted a deeper analysis before committing to Phase 3.

**Question posed**: Is the "hybrid curated wiki + RAG" architecture the right choice? Should we consider GraphRAG (Microsoft's knowledge graph RAG) or Agentic RAG instead?

**Sources ingested**:
1. `GraphRAG_ A new approach for discovery using complex information.md` — Microsoft Research Blog introducing GraphRAG; VIINA dataset demo showing GraphRAG vs baseline RAG
2. `GraphRAG_Microsoft introduction.md` — Official GraphRAG documentation; architecture overview, query modes (Global, Local, DRIFT, Basic), indexing process
3. `A modular Agentic RAG built with LangGraph — learn Retrieval-Augmented Generation Agents in minutes..md` — Full LangGraph Agentic RAG implementation; hierarchical indexing, query rewriting, hybrid search, orchestrator loop
4. `RAG and Agent - Snippet from the book - AI Engineering by Huyen.pdf` — Chapters on RAG architecture, agents, memory systems; key insight on compound error risk

**Web research conducted**: GraphRAG performance on structured educational content; comparison studies.

## Decisions Made

### GraphRAG: Rejected
- GraphRAG is designed for unstructured narrative data (news, social media, enterprise comms)
- A 2024 study on an undergraduate math textbook found standard RAG outperforms GraphRAG on retrieval accuracy and F1 — GraphRAG retrieves too much content on structured documents
- Our manually curated wiki with wiki-links IS a knowledge graph, but with human accuracy instead of LLM extraction noise
- Brittle to updates: curriculum changes require re-indexing large graph portions

### Full Agentic RAG: Rejected
- Compound error risk: 95% accuracy/step → 60% success over 10 steps → 0.6% over 100 steps (Huyen)
- Most student queries are direct ("what is Newton's second law?") and don't benefit from multi-step agent loops
- High-stakes exam prep demands predictability; agent orchestration adds unpredictability

### Architecture confirmed: Hybrid curated wiki + Upgraded RAG
Original decision stands, now with stronger evidence and a more specific implementation target.

**RAG layer upgrade plan** (borrowing from Agentic RAG research, without agent complexity):
- Hierarchical indexing: parent chunks (by wiki section) + child chunks (small, for precise search)
- Hybrid search: dense embeddings + BM25, merged with Reciprocal Rank Fusion (RRF)
- Query rewriting: rewrite colloquial Thai student queries before embedding search
- Contextual retrieval: add 50–100 token AI-generated context to each chunk at index time (Anthropic technique)
